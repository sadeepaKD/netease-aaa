// ABOUT THE PROJECT

Securing Campus Networks
the Smart Way

Small-scale organizations like university departments, SMEs, and school labs often rely on flat network architectures with shared credentials — leaving them vulnerable to unauthorized access and lacking visibility into who connects to their network.

NetEase AAA addresses this by providing a complete, deployable solution that brings enterprise-grade network authentication and monitoring to organizations that lack the budget for commercial solutions like Cisco ISE or Aruba ClearPass.

Built around the AAA framework (Authentication, Authorization, and Accounting), the system uses industry-standard protocols including IEEE 802.1X, RADIUS, and IPsec VPN to provide centralized user management with per-user VLAN assignment, real-time switch monitoring, and a modern web-based dashboard — all running on low-cost infrastructure via Docker containers.

// THE PROBLEM

What Happens Without AAA?

Shared Credentials

Everyone uses the same Wi-Fi password or plugs into any port. No individual accountability. If credentials leak, the entire network is compromised.

Flat Networks

All devices share one broadcast domain. A student's laptop sits on the same segment as the department's printers and servers — no segmentation or isolation.

Zero Visibility

Administrators have no idea who is connected, when they connected, or what they accessed. Troubleshooting and incident response become guesswork.

// KEY FEATURES

What NetEase AAA Delivers

01

802.1X Port Authentication

Every switch port requires individual user credentials before granting network access. Unauthorized devices are blocked at the physical layer.

02

Dynamic VLAN Assignment

Users are automatically placed into their assigned VLAN upon authentication. Staff, students, and guests get isolated network segments without manual port configuration.

03

Centralized User Management

Create, edit, enable/disable users from a single web dashboard. Assign roles and VLANs. Changes take effect immediately — no switch CLI needed.

04

Real-time Switch Monitoring

Live dashboard showing port status, VLAN assignments, CPU/memory utilization, connected MAC addresses, and interface statistics for every switch port.

05

Authentication Logging

Full audit trail via RADIUS accounting. Track who connected, when, from which port, session duration, and authentication success/failure history.

06

Cloud-to-Lab VPN

Secure IPsec tunnel connects the cloud management plane to the on-premise switch. All RADIUS traffic and SSH monitoring flows through encrypted tunnel.

// HOW IT WORKS

The Authentication Flow

1

User Connects

A user plugs their laptop into a switch port. The port is in an unauthorized state — no traffic is allowed through except 802.1X EAPOL frames.

Laptop EAP Switch Port
2

Credentials Prompted

The operating system's 802.1X supplicant prompts for username and password. The switch acts as an authenticator and forwards credentials to the RADIUS server.

Switch RADIUS FreeRADIUS
3

Database Lookup

FreeRADIUS queries the MariaDB database to verify the credentials. It checks the user's password, status (enabled/disabled), role, and VLAN assignment.

RADIUS SQL MariaDB
4

Access Granted + VLAN

If credentials are valid, RADIUS sends an Access-Accept with VLAN attributes. The switch places the port into the assigned VLAN and the user gets full network access.

RADIUS Accept Switch → VLAN
